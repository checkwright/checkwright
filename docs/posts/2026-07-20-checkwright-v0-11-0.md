# Checkwright v0.11.0

*2026-07-20*

Checkwright is the verification layer under agent orchestration, and this release
places lifecycle rules where the sessions that need them will read them: work
enters only through the scope gate, the spec stage now routes design-ruling debt
rather than features alone, and close states where its own captures file.

## Tightened gates

- **check-doctrine-registration** — a new always-loaded methodology rule,
  *Scope-gated intake*, landed in `doctrine-kit/DOCTRINE.md` with its digest
  bullet in `CLAUDE.md`, and the gate holds the two in lockstep. A consumer who
  re-vendors the doctrine kit but has not added the rule's digest line to their
  own `CLAUDE.md` reds here; register the bullet to clear it.

## Renamed knobs

None.

## Behavior changes

- **doctrine-kit/DOCTRINE.md** — the new *Scope-gated intake* rule: a new
  initiative raised mid-session is filed as a costed Deferred entry by default,
  never started, entering active work only through the scope gate. It is Gap
  disposition's intake-side sibling — guarding over-action on a raised want as
  that rule guards under-action on a found gap.
- **lifecycle-kit/templates/skills/spec.md** — the `spec`-stage trigger widened
  from feature-only to any unit carrying authoring. A debt unit whose design is a
  ruling (a placement, seam, or among-alternatives decision) now routes through
  `spec`, which lands it directly in the governed surface with no amendment and
  no `[spec:]` tag; a no-build debt ruling makes its Done move at `spec` and
  recommends `validate`, skipping build.
- **lifecycle-kit/templates/skills/scope.md** — scope now weighs the iteration's
  cost before opening it (the economic composition test — bundle a
  sub-threshold unit's shared-surface siblings or argue it stands alone), and its
  next-stage routing recommends the authoring stage for a design-ruling debt
  unit, not features alone.
- **lifecycle-kit/templates/skills/close.md** — close now states where its own
  captures file: every finding close itself captures (a lesson turned task, a
  drained gap, a triage finding) files as a Deferred `[needs-spec]` entry, never
  into the active queue sections.

## Upgrading

Replace the vendored kit directories wholesale at `v0.11.0`, then regenerate the
generated artifacts — the pre-commit hook via `gate-sdk/bin/gen-pre-commit.sh
--write` and the graph projection via `gate-sdk/checks/check-graph.sh --emit` —
and run the full battery.

**The allowed red.** Tightened gates is non-empty, so a clean upgrade may turn
`check-doctrine-registration` red: it reds when your vendored
`doctrine-kit/DOCTRINE.md` carries the new *Scope-gated intake* rule but your
`CLAUDE.md` digest does not yet register it. Add the digest bullet naming the
rule, and the gate goes green.

The behavior changes above are declared for reading, not a mechanical scan. If a
gate reds that this note does not name, the upgrade smoke was supposed to catch
it first — [open an issue](https://github.com/checkwright/checkwright/issues),
because that is a defect in the release rather than work for you.
